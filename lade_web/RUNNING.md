# LaDe Web — Running Guide

## TL;DR

This project has **two parts**:

| Part | What it does | When needed |
|------|--------------|-------------|
| **Frontend** (`index.html` + `app.js`) | The whole web UI + 5 of the 6 tabs | Always |
| **Backend** (`backend.py`) | Forwards questions to DeepSeek API | Only for the LLM tab |

**During your presentation you need BOTH running.** If you skip the backend,
the LLM tab still works — it just falls back to an offline cache and shows
"cached" in the result status.

---

## First-time setup (do once)

### 1. Frontend dependencies

None — the frontend is static HTML + vanilla JS. It loads `sql.js` and
`Chart.js` from CDN on first open.

### 2. Backend dependencies

Open a terminal, `cd` into the project folder, then:

```bash
pip3 install -r requirements.txt
```

This installs `flask`, `flask-cors`, `openai`, `python-dotenv` (~10 MB total).

### 3. Get a DeepSeek API key

1. Go to <https://platform.deepseek.com/>
2. Sign up (no credit card required)
3. Visit the **API Keys** page and click **Create API Key**
4. Copy the key — it starts with `sk-...` — you won't be able to see it again

DeepSeek gives new accounts **5 million free tokens**. Our demo uses
roughly 800 tokens per question, so that's ~6,000 free questions. Plenty.

### 4. Put the API key in `.env`

In the project folder:

```bash
cp .env.example .env
```

Then open `.env` in any text editor and replace the placeholder:

```
DEEPSEEK_API_KEY=sk-your-actual-key-here
```

**Never commit `.env` to Git.** The `.gitignore` should already exclude it.

## Running the project

You need **two terminals** open at the same time.

### Terminal 1 — Backend

```bash
cd path/to/lade_web
python3 backend.py
```

You should see:

```
============================================================
LaDe LLM backend running on http://localhost:5001
Keep this terminal open during your presentation.
============================================================
 * Running on http://127.0.0.1:5001
```

**Leave this terminal open.** Closing it stops the backend.

### Terminal 2 — Frontend

Two options:

**Option A · VS Code Live Server** (recommended)
1. Open the project folder in VS Code
2. Right-click `index.html` → "Open with Live Server"

**Option B · Python's built-in server**
```bash
cd path/to/lade_web
python3 -m http.server 8000
```
Then open <http://localhost:8000/> in your browser.

## How to tell if it's working

Open the **"Ask in English"** tab and click any example question.

- Status shows **`✓ N rows · XX ms · live`** → backend is reachable,
  SQL was generated by DeepSeek in real time.
- Status shows **`✓ N rows · XX ms · cached`** → backend is unreachable,
  the frontend fell back to the offline cache.
- Status shows **`✗ Backend offline + no cache match`** → backend is down
  *and* the question doesn't match any cached entry. Start the backend
  or rephrase.

## Troubleshooting

**"Address already in use" when starting backend**
Another process is holding port 5001. Kill it with:
```bash
lsof -ti:5001 | xargs kill -9     # macOS / Linux
```
Or change the port in `backend.py` (last line) and update `BACKEND_URL` in `app.js`.

**"Module not found: flask"**
You forgot `pip3 install -r requirements.txt`. Run it.

**"DEEPSEEK_API_KEY not found"**
The `.env` file is missing or the key line is wrong. Make sure the file
is in the same folder as `backend.py` and contains:
```
DEEPSEEK_API_KEY=sk-...
```
with no quotes or spaces around the `=`.

**"LLM call failed: ... 401"**
Your API key is invalid or the account has no credits. Create a new
key at <https://platform.deepseek.com/>.

**Browser console shows `CORS` error**
The `flask-cors` package isn't installed or the `CORS(app)` line in
`backend.py` got removed. Reinstall from `requirements.txt`.

## What to demonstrate during the presentation

1. **Start both terminals** before your talk begins.
2. Open the "Ask in English" tab.
3. Type a question the audience suggests — show the `live` tag in the status.
4. **Kill Terminal 1** (Ctrl+C the backend), ask another question — show
   the seamless fallback to `cached`. This is a great talking point for
   your "fault-tolerant design" section.
5. Restart the backend with `python3 backend.py` to return to live mode.

Bullet for your report:

> "The system uses a two-layer LLM integration: primary path calls DeepSeek
> V3.2 via a Flask proxy on localhost:5001; if the API is unreachable, the
> frontend transparently falls back to an offline cache of curated question/
> SQL pairs. This ensures 100% availability during live demonstrations."
